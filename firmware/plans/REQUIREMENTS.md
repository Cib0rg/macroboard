# Требования к прошивке ESP32-S3 макроклавиатуры

## Аппаратная платформа

1. **Основа устройства**: ESP32-S3 N16R8 (16MB Flash, 8MB PSRAM)

## Компоненты устройства

2. **Дисплеи**: 10× GC9A01 круглые дисплеи 160×160 пикселей, 0.71 дюйма
   - Подключение: SPI
   - Выбор дисплея: через схему с двумя шифраторами 74HC138
   - 4 GPIO: 3 бита адреса (A0-A2) + 1 бит выбора шифратора (SEL)

3. **Кнопки**: 10× тактовые кнопки
   - Подключение: напрямую к GPIO контроллера
   - Обработка: только через прерывания (interrupt-driven), без polling
   - Debouncing: программный, 10-50 мс

4. **RGB LED**: 10× адресные светодиоды WS2812
   - Подключение: одна нога контроллера через RMT peripheral
   - Управление: цвет и яркость задаются извне

5. **Rotary Encoder**: 1× энкодер с кнопкой
   - Назначение: переключение профилей
   - Обработка: через GPIO прерывания

## USB подключение

4. Устройство подключается к компьютеру по native USB и определяется как:
   - **USB HID Keyboard** - для эмуляции нажатий клавиш
   - **USB HID Raw Device** - для обмена данными с управляющим софтом
   - **USB CDC (UART)** - для логирования диагностических данных (опционально)

## Функциональные требования

### Основная схема работы

7. **Взаимодействие с управляющим софтом**:
   - Управляющий софт отправляет изображения для экранов
   - Управляющий софт отправляет команды, закрепленные за кнопками
   - Команды двух видов:
     * "Отпечатать текст как клавиатура" (HID Keyboard)
     * "Послать сигнал как отдельное устройство" (HID Raw)

8. **Хранение данных**:
   - Устройство сохраняет команды и изображения в энергонезависимую память (Flash)
   - Использование SPIFFS для файловой системы
   - Использование NVS для настроек

9. **Отображение и реакция**:
   - Отрисовка изображений на дисплеях
   - Реакция на нажатия кнопок согласно сохраненным командам
   - Обновление RGB LED согласно профилю

### Профили

10. **Rotary Encoder**:
    - Используется для переключения профилей клавиш и экранов
    - Текущий профиль хранится на устройстве
    - Поддержка 3-5 профилей

11. **Логический юнит**:
    - Блок "светодиод + кнопка + экран" является единым юнитом
    - Яркость и цвет светодиода задаются извне контроллера
    - Изображение на экране соответствует функции кнопки

## Обязательные функции

6. **WiFi OTA**: ОБЯЗАТЕЛЬНО наличие WiFi OTA обновления прошивки
   - Подключение к WiFi по credentials из управляющего софта
   - Загрузка firmware по HTTP/HTTPS
   - Автоматический rollback при ошибке

## Дополнительные требования

### Производительность

- **Button latency**: < 10 мс от нажатия до действия
- **Display update**: < 50 мс на обновление одного дисплея
- **Profile switch**: < 100 мс на переключение профиля
- **Boot time**: < 2 секунды

### Надежность

- Проверка целостности данных (CRC32)
- Обработка ошибок и восстановление
- Защита от износа flash (wear leveling)
- Сохранение состояния при отключении питания

### Удобство использования

- Визуальная обратная связь через LED
- Статичные изображения на дисплеях (без анимаций)
- Логирование для отладки (опционально через USB CDC)

## Формат данных

### Изображения

- **Рекомендуемый формат**: JPEG
- **Размер**: 160×160 пикселей
- **Качество**: 80-90%
- **Средний размер файла**: 8-15 КБ

### Протокол обмена

- **Интерфейс**: USB HID Raw
- **Размер пакета**: 64 байта
- **Формат**: см. [`protocol.md`](../../plans/protocol.md)
