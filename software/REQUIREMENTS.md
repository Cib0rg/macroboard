# Требования к управляющему софту для макроклавиатуры

## Обзор

Управляющее приложение для настройки ESP32-S3 макроклавиатуры с графическим интерфейсом, аналогичное софту Mad Catz для игровых периферийных устройств.

**ВАЖНО**: Приложение должно быть совместимо с плагинами Elgato Stream Deck через эмуляцию WebSocket API.

## Платформа и технологии

### Рекомендуемый стек

- **Язык**: C# (.NET 8.0+)
- **UI Framework**: 
  - **Вариант 1**: WPF (Windows Presentation Foundation) - для Windows
  - **Вариант 2**: Avalonia UI - кроссплатформенный (Windows, macOS, Linux)
  - **Вариант 3**: WinUI 3 - современный Windows UI
- **Архитектура**: MVVM (Model-View-ViewModel) + Plugin System
- **USB Communication**: HidLibrary или LibUsbDotNet
- **Image Processing**: System.Drawing или ImageSharp
- **Plugin System**: WebSocket Server (совместимый с Elgato Stream Deck API)
- **Dependency Injection**: Microsoft.Extensions.DependencyInjection

## Функциональные требования

### 1. Обнаружение и подключение устройства

#### FR-1.1: Автоматическое обнаружение
- Приложение автоматически обнаруживает подключенное устройство по VID/PID
- Отображение статуса подключения в UI
- Уведомление при подключении/отключении устройства
- Поддержка hot-plug (подключение во время работы приложения)

#### FR-1.2: Информация об устройстве
- Отображение версии прошивки
- Отображение серийного номера
- Отображение свободного места в памяти
- Отображение текущего профиля

### 2. Управление профилями

#### FR-2.1: Создание и редактирование профилей
- Создание нового профиля с именем
- Редактирование существующего профиля
- Удаление профиля
- Копирование профиля
- Импорт/экспорт профиля в файл

#### FR-2.2: Переключение профилей
- Выбор активного профиля из списка
- Отправка профиля на устройство
- Синхронизация с устройством (загрузка профилей с устройства)

#### FR-2.3: Ограничения
- Поддержка 3-5 профилей (согласно прошивке)
- Валидация имени профиля (длина, допустимые символы)

### 3. Настройка кнопок

#### FR-3.1: Визуальное представление
- Отображение 10 кнопок в виде сетки или круга
- Визуальная индикация настроенных/ненастроенных кнопок
- Предпросмотр изображения на кнопке
- Предпросмотр цвета LED

#### FR-3.2: Настройка действия кнопки
**Тип 1: Keyboard Action**
- Выбор клавиши или комбинации клавиш (Ctrl, Shift, Alt, GUI + клавиша)
- Ввод текста для печати
- Поддержка специальных клавиш (F1-F24, медиа-клавиши)
- Макросы (последовательность нажатий с задержками)

**Тип 2: Custom HID Action**
- Отправка пользовательского HID report
- Hex-редактор для ввода данных
- Шаблоны для популярных действий

#### FR-3.3: Настройка изображения
- Загрузка изображения из файла (PNG, JPG, BMP, GIF)
- Автоматическое масштабирование до 160×160
- Обрезка изображения (crop tool)
- Библиотека предустановленных иконок
- Редактор изображений (базовый):
  * Добавление текста
  * Выбор цвета фона
  * Простые фигуры

#### FR-3.4: Настройка RGB LED
- Выбор цвета через color picker
- Настройка яркости (0-100%)
- Выбор эффекта:
  * Static (статичный цвет)
  * Breathing (дыхание)
  * Rainbow (радуга)
- Предпросмотр эффекта

### 4. Модуль повседневной работы (Tray Application)

#### FR-4.1: Системный трей
- Иконка в system tray
- Минимизация в трей
- Быстрый доступ к функциям:
  * Переключение профиля
  * Открытие настроек
  * Выход

#### FR-4.2: Мониторинг устройства
- Отслеживание подключения устройства
- Отображение текущего профиля
- Уведомления о событиях (смена профиля, ошибки)

#### FR-4.3: Горячие клавиши
- Глобальные горячие клавиши для переключения профилей
- Настройка горячих клавиш пользователем

### 5. Модуль настройки (Configuration Application)

#### FR-5.1: Главное окно
- Список профилей слева
- Область настройки кнопок в центре
- Панель свойств справа
- Toolbar с основными действиями

#### FR-5.2: Workflow настройки
1. Выбор профиля
2. Выбор кнопки для настройки
3. Настройка действия, изображения, LED
4. Сохранение изменений
5. Отправка на устройство

#### FR-5.3: Предпросмотр
- Реалтайм предпросмотр изменений
- Тестирование действий без отправки на устройство
- Визуализация профиля целиком

### 6. WiFi и OTA обновление

#### FR-6.1: Настройка WiFi
- Ввод SSID и пароля
- Отправка credentials на устройство
- Проверка подключения
- Отображение статуса WiFi (подключено/отключено, IP адрес, сигнал)

#### FR-6.2: OTA обновление
- Выбор файла прошивки (.bin)
- Загрузка прошивки на HTTP сервер (встроенный или внешний)
- Отправка команды OTA на устройство
- Отображение прогресса обновления
- Уведомление об успехе/ошибке

#### FR-6.3: Проверка обновлений
- Автоматическая проверка обновлений прошивки
- Уведомление о доступных обновлениях
- Скачивание и установка обновлений

### 7. Система плагинов (Stream Deck совместимость)

#### FR-7.1: Backend Service
- Работа в фоне как Windows Service или системный процесс
- WebSocket сервер на порту 28196 (стандартный для Stream Deck)
- Эмуляция Stream Deck WebSocket API
- Маршрутизация событий между устройством и плагинами

#### FR-7.2: Загрузка плагинов
- Автоматическое обнаружение плагинов в директории
- Загрузка манифеста плагина (manifest.json)
- Запуск плагинов (Node.js, Python, или .NET)
- Управление жизненным циклом плагинов

#### FR-7.3: Plugin Browser
- Просмотр установленных плагинов
- Установка новых плагинов (из файла или репозитория)
- Удаление плагинов
- Обновление плагинов
- Включение/отключение плагинов

#### FR-7.4: Совместимость с Stream Deck плагинами
- Поддержка Stream Deck WebSocket API
- Адаптация изображений (72×72 → 160×160, квадрат → круг)
- Эмуляция device info (тип устройства, размер)
- Маппинг координат кнопок

#### FR-7.5: Встроенные плагины
- **Keyboard Plugin** - эмуляция клавиш
- **Text Plugin** - отображение текста
- **Image Plugin** - статичное изображение
- **Clock Plugin** - часы
- **System Plugin** - системная информация (CPU, RAM)
- **Multi-Action Plugin** - последовательность действий

#### FR-7.6: Безопасность плагинов
- Изоляция плагинов (sandboxing)
- Ограничение доступа к файловой системе
- Валидация манифеста
- Проверка подписи плагинов (опционально)

### 8. Диагностика и отладка

#### FR-7.1: Логирование
- Просмотр логов с устройства (через USB CDC)
- Фильтрация логов по уровню (Error, Warning, Info, Debug)
- Сохранение логов в файл
- Очистка логов

#### FR-7.2: Тестирование
- Тест кнопок (отображение нажатий)
- Тест дисплеев (отправка тестовых изображений)
- Тест LED (проверка цветов и эффектов)
- Тест энкодера

#### FR-7.3: Информация о системе
- Версия приложения
- Версия прошивки устройства
- Использование памяти устройства
- Статистика (количество нажатий, время работы)

## Нефункциональные требования

### NFR-1: Производительность
- Запуск приложения < 3 секунд
- Отклик UI < 100 мс
- Отправка профиля на устройство < 5 секунд
- Обновление изображения на кнопке < 500 мс

### NFR-2: Удобство использования
- Интуитивный интерфейс (не требует обучения)
- Drag & drop для изображений
- Undo/Redo для действий
- Контекстная помощь (tooltips)
- Keyboard shortcuts для основных действий

### NFR-3: Надежность
- Обработка ошибок подключения
- Валидация данных перед отправкой
- Автосохранение изменений
- Резервное копирование профилей
- Восстановление после сбоя

### NFR-4: Безопасность
- Проверка целостности данных (CRC)
- Валидация прошивки перед OTA
- Безопасное хранение WiFi credentials (шифрование)

### NFR-5: Расширяемость
- Модульная архитектура
- Плагины для дополнительных функций
- API для интеграции с другими приложениями
- Поддержка тем оформления

### NFR-6: Локализация
- Поддержка нескольких языков (EN, RU)
- Возможность добавления новых языков
- Локализация дат, времени, чисел

## Технические детали

### USB HID Communication

```csharp
// Пример использования HidLibrary
using HidLibrary;

public class DeviceService
{
    private const int VendorId = 0x303A;  // Espressif
    private const int ProductId = 0x4001; // Наш продукт
    
    public HidDevice FindDevice()
    {
        var devices = HidDevices.Enumerate(VendorId, ProductId);
        return devices.FirstOrDefault();
    }
    
    public async Task<byte[]> SendCommandAsync(byte[] command)
    {
        // Отправка команды и получение ответа
    }
}
```

### Протокол обмена

См. [`../plans/protocol.md`](../plans/protocol.md) для детального описания протокола.

Приложение должно реализовать все команды протокола:
- PING, GET_DEVICE_INFO
- SET_PROFILE, GET_PROFILE_INFO
- START_IMAGE_TRANSFER, IMAGE_DATA_CHUNK, END_IMAGE_TRANSFER
- SET_BUTTON_ACTION, SET_LED_COLOR
- SAVE_PROFILE, LOAD_PROFILE, DELETE_PROFILE
- START_OTA_UPDATE, GET_OTA_STATUS
- SET_WIFI_CREDENTIALS, GET_WIFI_STATUS

### Хранение данных

**Локальное хранилище**:
- Профили: JSON файлы в `%AppData%/MacroKeyboard/Profiles/`
- Изображения: PNG файлы в `%AppData%/MacroKeyboard/Images/`
- Настройки: JSON файл в `%AppData%/MacroKeyboard/settings.json`
- Логи: текстовые файлы в `%AppData%/MacroKeyboard/Logs/`

**Формат профиля**:
```json
{
  "id": 0,
  "name": "Gaming",
  "buttons": [
    {
      "id": 0,
      "actionType": "Keyboard",
      "actionData": {
        "modifiers": ["Ctrl"],
        "key": "C"
      },
      "imagePath": "images/copy.png",
      "led": {
        "r": 255,
        "g": 0,
        "b": 0,
        "brightness": 128,
        "effect": "Static"
      }
    }
  ]
}
```
